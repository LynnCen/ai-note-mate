# 需求文档

## 项目名称

Web 端 AI 笔记应用

---

## 1. 项目背景

你的任务是构建一个微型的、AI 驱动的笔记应用，用于帮助用户快速记录和「润色」灵感。

用户能在一个地方写下他们的想法，并能随时调用 AI 来帮助他们扩展、重写或总结这些想法。我们追求的目标是实现一个双端可用的实时、流畅且对用户友好的交互体验。

---

## 2. 核心功能需求

你必须实现以下所有功能：

### 2.1 基础笔记功能

- 应用需要包含一个笔记列表视图和一个笔记详情视图。
- 用户必须能够创建新笔记、编辑笔记内容和删除笔记。
- 笔记数据必须在浏览器端持久化，确保刷新页面后内容不丢失。
  - **加分项**：如果你用服务端 + 数据库来实现，会有加分。
- **加分项**：如果使用 Firebase 或 WebSocket 实现笔记的实时协作编辑。
- **加分项**：使用一个企业级状态库重构应用，以支持未来的「分类」和「标签」功能。
- **加分项**：解决当笔记文本超长（万行）时，AI 流式打字卡顿掉帧的问题。
- **加分项**：为 AI 交互的核心逻辑（如流处理、悬浮框组件）编写单元和组件测试。

### 2.2 AI 助手交互

#### 触发机制

- 在笔记详情页，需要有一个「AI 处理」按钮（或类似功能）。
- 当用户高亮选中了笔记中的部分文本时，点击该按钮应只处理被选中的文本。
- 当用户没有选中任何文本时，点击该按钮则应处理整篇笔记的全部内容。

#### AI 响应与交互

- 触发 AI 后，应弹出一个悬浮框（Modal/Popover），AI 的响应必须以「流式（Streaming）」的方式（即逐字显示）出现在这个悬浮框中。
- 在此期间，原笔记内容不应被改动。
- 悬浮框中必须提供「接受」和「丢弃」两个选项。
- 点击「接受」：用悬浮框中的 AI 新内容替换原笔记内容（或被选中的那部分内容）。
- 点击「丢弃」：关闭悬浮框，原笔记内容保持不变。

### 2.3 API 服务与配置

- 你可以自由选择任何提供免费额度的 LLM（大语言模型）API 来实现此功能。
- 我们希望这个项目对于使用者（即面试官）是「开箱即用」或「易于配置」的。请你妥善处理 API Key 的配置问题，确保它足够安全并考虑到最终的使用体验。

## 3. 工程标准要求

1. **TypeScript**：你的项目必须使用 TypeScript，并尽可能不使用 `any`。
2. **CSS 作用域化**：你必须使用一种能确保组件样式隔离的方案，以避免全局 CSS 污染。
3. **现代框架**：你必须使用一个现代前端框架或库（例如 React、Vue）来构建此应用。
4. **基础用户体验**：用户在使用过程中不应该出现白屏、crash 等完全无法使用应用的情况，在使用过程中用户体验应该得到保障。
5. **响应式设计**：你的应用必须在桌面端和主流移动设备宽度（例如 375px ~ 420px）上都具有良好的布局和可用性。
